CheatSheet

The following provides a basic comparison between the Ruby and JavaScript programming languages. It is a work in progress, far from complete, and will be updated as I learn.

Ruby

Syntax: Keep it simple.

Ruby syntax is extremely simple. The programmer need type little more than absolutely necessary.

  • No declaration of any kind is necessary to declare variables. One simply initializes it: 
    variable = 27
  • Methods are not treated as variables (unless they are lambdas). They are given their own syntax: 
    def function(input)
  input * 2
end
  • Hashes and arrays are predefined object types.

JavaScript

Syntax: Keep it slightly simpler than Java.

JavaScript's syntax is like a simplified version of Java. Which is to say, not very simple.

  • Variables must be declared as variables, but no type need be specified:
    var variable = 27;
  • Most lines must end with a semicolon. JavaScript is not as strict about this requirement as Java, but it makes life easier to just assume they are needed in the same places.
  • Functions can be treated as variables or not, depending on the preference of the programmer.
    • If not treated as a variable, the function will be compiled prior to runtime. This means that the function can be declared anywhere in the code.
      function name(input{
  input * 2;
}
    • If a function is treated as a variable, it must be declared in code before it can be used.
      var name = function(input){
  input * 2;
}
  • There is no such thing as a hash in JavaScript because an object is a hash.
  • Array are defined as a special class of object.

Defining a new object class

In ruby, everything is an object, so creating new ones is nothing special. However, one can customize the process by creating classes of objects that contain certain properties and functions.

  • Classes are created entirely inside the class designator.
  • A class must contain an explicit initialization function or constructor.
  • Instance variables are identified with a preceding & symbol.
  •                     class Sample
                      def initialize(prop1, prop2)
                        @prop1 = prop1
                        @prop2 = prop2
                      end
                      def sum
                        @prop1 + @prop2
                      end
                    end

Creating a new object prototype

In JavaScript, one does not define a class, but create a constructor function, and then modify the prototype object used by that function to create instance objects.

  • Classes are defined by writing constructors, which are functions.
  • No initialization function need be defined because the class definition is itself the constructor function.
  • The constructor does not define methods for that class. These must be added afterwards.
  • In JavaScript, all objects have a 'prototype', which is like a blueprint object. Class methods are created by modifying the class prototype.
  • Instance variables are identified with the 'this' prefix, which is similar to 'self' in ruby.
  •                       function Sample(prop1, prop2){
                        this.prop1 = prop1;
                        this.prop2 = prop2;
                      }

                      Sample.prototype.sum = function(){
                        return this.prop1 + this.prop2;
                      }
